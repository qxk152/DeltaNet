多头注意力与 Transformer Block

上一课我们学了单头自注意力。实际模型使用多头注意力—— 把大向量拆成多个小头并行计算，每个头关注不同的模式。 然后我们把所有积木组装成一个完整的 Transformer Block—— 这正是 GDN 层所在的上下文。

多头注意力：让模型"多角度"看序列

直觉

一个注意力头只能学一种关注模式。比如：

  • 头1可能学习语法关系（主语-谓语）
  • 头2可能学习语义关系（同义词）
  • 头3可能学习位置关系（相邻 token）

多头 = 并行运行多个注意力头，每个头看不同维度。

实现方式

不是真的创建多个小矩阵，而是把 hidden_size 拆分成 num_heads × head_dim：

# hidden_size = 512, num_heads = 8, head_dim = 64
# 一个头的维度 = 64

# 投影后的形状变化:
x:     [B, T, 512]       # 输入
q:     [B, T, 512]       # 投影后还是 512
q:     [B, T, 8, 64]     # 拆成 8 个头，每个 64 维
q:     [B, 8, T, 64]     # 转置，让头维度在前

# 每个头独立做注意力 (64 维向量之间)
# 8 个头的结果拼回 512 维

图示

          输入 x [B, T, 512]
              │
     ┌────────┼────────┐
     │        │        │
   Q投影    K投影    V投影
     │        │        │
  [B,T,512] [B,T,512] [B,T,512]
     │        │        │
  拆成8头  拆成8头  拆成8头
     │        │        │
  [B,8,T,64] [B,8,T,64] [B,8,T,64]
     │        │        │
     └────注意力(每个头独立)────┘
              │
         [B, 8, T, 64]
              │
         拼回 [B, T, 512]
              │
           O投影
              │
         [B, T, 512]
关键：每个头看到的是同一输入的不同投影子空间。 头的数量 × 头的维度 = 隐藏维度。例如 GDN 默认 num_heads=6, head_dim=256，所以 key_dim = 6×256 = 1536。

GDN 的多头差异

GDN 与标准注意力在多头上的一个重要差异：K 和 V 的维度可以不同。

# 标准注意力:
#   head_dim = head_k_dim = head_v_dim (K和V同维度)

# GDN:
#   head_k_dim = 256  (key 维度)
#   head_v_dim = 512  (value 维度, expand_v=2)
#   状态矩阵 S: [256, 512] 而非 [256, 256]

这意味着 GDN 的状态矩阵可以是"长方形"的——用较小的 key 维度检索，存储较大的 value 维度信息。

FFN / MLP 层：逐位置的非线性变换

注意力层让 token 之间交互信息。之后需要一个逐位置的前馈网络 (FFN)， 对每个 token 独立做非线性变换。现代模型使用 SwiGLU：

$$\text{FFN}(x) = (\text{SiLU}(x W_{\text{gate}}) \odot x W_{\text{up}}) W_{\text{down}}$$

三个线性层：gate 和 up 扩展维度（通常 4×），down 压缩回来。

class SwiGLU(nn.Module):
    def __init__(self, hidden_size, intermediate_size):
        super().__init__()
        self.gate = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.up   = nn.Linear(hidden_size, intermediate_size, bias=False)
        self.down = nn.Linear(intermediate_size, hidden_size, bias=False)

    def forward(self, x):
        return self.down(F.silu(self.gate(x)) * self.up(x))

完整 Transformer Block

把所有积木拼起来。一个 Pre-Norm Transformer Block：

class TransformerBlock(nn.Module):
    def __init__(self, hidden_size, num_heads):
        super().__init__()
        self.attn_norm = nn.RMSNorm(hidden_size)    # 注意力前归一化
        self.attn = SelfAttention(hidden_size, num_heads)
        self.mlp_norm = nn.RMSNorm(hidden_size)     # MLP前归一化
        self.mlp = SwiGLU(hidden_size, hidden_size * 4)

    def forward(self, x):
        # 子层1: Pre-Norm + 注意力 + 残差
        x = x + self.attn(self.attn_norm(x))

        # 子层2: Pre-Norm + MLP + 残差
        x = x + self.mlp(self.mlp_norm(x))
        return x

GDN Block：唯一的不同

GDN 的 Block 结构完全一样，只是注意力层被替换：

组件 标准 Transformer GDN
注意力层 SelfAttention GatedDeltaNet
归一化 RMSNorm RMSNorm (相同)
MLP SwiGLU SwiGLU (相同)
残差连接 Pre-Norm Pre-Norm (相同)
短卷积 无 Q/K/V 各一个 ShortConv
理解到这里：你已经知道了一个 Transformer/GDN Block 的完整结构。 后续课程将深入 GatedDeltaNet 层的内部—— 那才是 GDN 的核心，也是你需要在昇腾上实现的部分。

完整的 GDN Block 代码

class GDNBlock(nn.Module):
    def __init__(self, hidden_size, num_heads, head_dim):
        super().__init__()
        self.attn_norm = RMSNorm(hidden_size)
        self.attn = GatedDeltaNet(
            hidden_size=hidden_size,
            num_heads=num_heads,
            head_dim=head_dim,
        )
        self.mlp_norm = RMSNorm(hidden_size)
        self.mlp = SwiGLU(hidden_size, hidden_size * 4)

    def forward(self, x):
        # 与标准 Transformer Block 完全相同的结构!
        x = x + self.attn(self.attn_norm(x))
        x = x + self.mlp(self.mlp_norm(x))
        return x

混合架构：GDN + 注意力

GDN 论文发现，纯 GDN 模型不如混合架构——大部分层用 GDN， 少量层用标准注意力。例如一个 24 层模型可能：

# 层 0-17: GDN (高效, 线性复杂度)
# 层 18-19: 滑动窗口注意力 (精确检索局部信息)
# 层 20-23: GDN
#
# GDN 负责长距离信息压缩, 注意力负责精确检索
# 兼顾效率和性能

这也是 Qwen3-Next 采用的策略——GDN 层 + 少量注意力层的混合。

测验

问题 1：多头注意力中，head_dim × num_heads 等于什么？
  • 序列长度
  • 层数
  • 隐藏维度 hidden_size
  • 词表大小
问题 2：Pre-Norm Transformer Block 的残差连接公式是？
  • x = norm(x + attn(x))
  • x = x + attn(norm(x))
  • x = attn(norm(x)) + norm(x)
  • x = norm(x) + attn(x)
问题 3：GDN Block 与标准 Transformer Block 的唯一结构差异是？
  • GDN 不使用残差连接
  • GDN 不使用 MLP 层
  • 注意力层被替换为 GatedDeltaNet 层（外加短卷积）
  • GDN 使用 LayerNorm 而非 RMSNorm
