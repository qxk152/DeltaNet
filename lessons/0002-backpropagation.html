反向传播与梯度下降

反向传播是深度学习的核心引擎。对 GDN 算子实现来说尤其重要——因为 自定义 kernel 需要手写反向传播，PyTorch 的 autograd 帮不了你。 这一课从零理解反向传播的工作原理。

梯度下降：从山顶下山

训练神经网络 = 找到一组参数（权重 $W$）使损失 $L$ 最小。梯度下降的做法：

$$W \leftarrow W - \eta \cdot \frac{\partial L}{\partial W}$$

$\eta$ 是学习率，$\frac{\partial L}{\partial W}$ 是损失对权重的梯度。 梯度指向"上坡"方向，所以减去梯度就是"下坡"。

核心问题：怎么算 $\frac{\partial L}{\partial W}$？当网络有很多层时， 损失是经过无数次变换才得到的，直接求导不可能。这就是反向传播要解决的。

前向传播：构建计算图

考虑一个简单的两层网络：

$$z_1 = W_1 x + b_1$$ $$a_1 = \sigma(z_1)$$ $$z_2 = W_2 a_1 + b_2$$ $$L = \text{loss}(z_2, y)$$

前向传播就是从左到右，一步步算出输出和损失。同时，这些操作构成一个计算图：

x → [W₁] → z₁ → [σ] → a₁ → [W₂] → z₂ → [loss] → L

链式法则：反向传播的数学基础

链式法则说：如果 $L$ 依赖于 $z_2$，$z_2$ 依赖于 $W_2$，那么：

$$\frac{\partial L}{\partial W_2} = \frac{\partial L}{\partial z_2} \cdot \frac{\partial z_2}{\partial W_2}$$

再往前一层：

$$\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial z_2} \cdot \frac{\partial z_2}{\partial a_1} \cdot \frac{\partial a_1}{\partial z_1} \cdot \frac{\partial z_1}{\partial W_1}$$
关键洞察：反向传播就是从损失开始，沿着计算图反向走， 每一步用链式法则把梯度传给上一层。每层只需要知道两件事：
  1. 从下游传回来的梯度 $\frac{\partial L}{\partial \text{output}}$
  2. 自己这层 output 对 input 的局部导数 $\frac{\partial \text{output}}{\partial \text{input}}$
两者相乘，就得到传给上一层的梯度。

为什么自定义 Kernel 需要手写反向？

PyTorch 的 autograd 能自动对标准操作（矩阵乘法、加法等）做反向传播。 但当你写了一个自定义 CUDA/Triton kernel时：

  • PyTorch 不知道 kernel 内部做了什么
  • 无法自动求导
  • 你必须手动推导梯度并写一个对应的反向 kernel
这对 GDN 算子意味着什么： GDN 的 chunk_gated_delta_rule kernel 包含复杂的递归状态更新。 在 NVIDIA 上，作者用 Triton 手写了前向和反向两个 kernel。 你要在昇腾上实现，就必须：
  1. 理解 GDN 前向计算的每一步
  2. 推导每一步的梯度如何回传
  3. 在 Ascend C++ 中实现前向和反向两个 kernel
这就是为什么理解反向传播对你至关重要。

矩阵运算的梯度：最常用的公式

以下是你在 GDN kernel 反向传播中会反复用到的核心公式：

前向 输入梯度 权重梯度
$Y = XW$ $\frac{\partial L}{\partial X} = \frac{\partial L}{\partial Y} W^\top$ $\frac{\partial L}{\partial W} = X^\top \frac{\partial L}{\partial Y}$
$Y = XW + b$ 同上 同上, $\frac{\partial L}{\partial b} = \sum \frac{\partial L}{\partial Y}$
$Y = \sigma(X)$ (逐元素) $\frac{\partial L}{\partial X} = \frac{\partial L}{\partial Y} \odot \sigma'(X)$ —
$Y = A \otimes B$ (外积) $\frac{\partial L}{\partial A} = \frac{\partial L}{\partial Y} B$ $\frac{\partial L}{\partial B} = A^\top \frac{\partial L}{\partial Y}$
最后一个公式尤其重要：GDN 的核心操作是状态更新 $S_{t} = \alpha S_{t-1} + k_t \otimes v_t$， 反向传播时需要对外积 $k_t \otimes v_t$ 求梯度——正是上面的公式。

  • (dL/dY) W
  • (dL/dY) W^T
  • W^T (dL/dY)
  • X^T (dL/dY)
问题 3：为什么自定义 kernel 需要手写反向传播？
  • 因为自动求导太慢
  • 因为 kernel 不支持浮点数
  • 因为 PyTorch 的 autograd 不知道 kernel 内部做了什么操作
  • 因为 kernel 不可微分
