自注意力机制

Gated DeltaNet 是自注意力的替代品。要理解 GDN 改进了什么， 必须先理解标准自注意力——它的工作原理、它的瓶颈、以及推理时的 KV Cache 问题。

核心直觉：每个 token "看"其他 token

对于序列中的每个 token，自注意力计算它对其他所有 token 的"关注程度"， 然后根据关注程度聚合信息。三个角色：

角色 符号 比喻
Query (Q) $q_t$ "我想知道什么" — 当前 token 的查询
Key (K) $k_i$ "我有什么" — 其他 token 的标签
Value (V) $v_i$ "我的内容" — 其他 token 的信息

计算步骤

第1步：投影

输入 $x \in \mathbb{R}^{N \times d}$ 通过三个线性层投影成 Q、K、V：

$$Q = x W_Q, \quad K = x W_K, \quad V = x W_V$$

第2步：计算注意力分数

每个 query 对每个 key 打分——点积越大越相关：

$$\text{scores} = \frac{Q K^\top}{\sqrt{d}} \in \mathbb{R}^{N \times N}$$

除以 $\sqrt{d}$ 是为了防止点积过大导致 softmax 饱和。

第3步：Softmax 归一化

把分数变成概率（和为1）：

$$A = \text{softmax}(\text{scores}) \in \mathbb{R}^{N \times N}$$

第4步：加权聚合

$$O = A V \in \mathbb{R}^{N \times d}$$

合在一起就是：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right) V$$

因果掩码：生成式模型的关键

语言模型生成文本时，每个 token 只能看到它之前的 token（不能偷看未来）。 这通过"因果掩码"实现——把上三角的注意力分数设为 $-\infty$：

O(N²) 瓶颈：为什么需要替代方案

注意力矩阵 $A \in \mathbb{R}^{N \times N}$ 是瓶颈的根源：

序列长度 N 注意力矩阵大小 显存 (float16)
2,048 4M 元素 ~8 MB
32,768 1B 元素 ~2 GB
131,072 16B 元素 ~32 GB
这就是 GDN 要解决的问题：用固定大小的状态矩阵（与 N 无关） 替代不断增长的注意力矩阵。代价是失去 softmax 的精确检索能力——GDN 通过门控和 delta 规则来弥补。

KV Cache：推理时的另一个问题

推理（生成）时，每生成一个新 token，需要计算它与所有历史 token 的注意力。 标准做法是缓存历史 K 和 V（KV Cache），避免重复计算：

# 生成第 N 个 token 时:
# 已经缓存了 k_1..k_{N-1}, v_1..v_{N-1}
# 新 token 产生 k_N, v_N

# 把新的加入缓存
k_cache.append(k_N)  # 缓存长度不断增长!
v_cache.append(v_N)

# 计算注意力（需要遍历所有历史）
scores = q_N @ k_cache.T   # O(N) 操作
attn = softmax(scores)
o_N = attn @ v_cache       # O(N) 操作
KV Cache 的代价：缓存大小 = $O(N \times d)$，随序列长度线性增长。 在长上下文场景下，KV Cache 的显存占用甚至超过模型本身。 GDN 用固定大小 $O(d^2)$ 的状态矩阵替代——这正是下一课（线性注意力）的主题。

PyTorch 实现：完整代码

import torch
import torch.nn as nn
import torch.nn.functional as F
from einops import rearrange

class SelfAttention(nn.Module):
    def __init__(self, hidden_size, num_heads):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = hidden_size // num_heads
        self.scale = self.head_dim ** -0.5

        self.q_proj = nn.Linear(hidden_size, hidden_size, bias=False)
        self.k_proj = nn.Linear(hidden_size, hidden_size, bias=False)
        self.v_proj = nn.Linear(hidden_size, hidden_size, bias=False)
        self.o_proj = nn.Linear(hidden_size, hidden_size, bias=False)

    def forward(self, x):
        B, T, C = x.shape  # batch, seq_len, hidden

        # 1. 投影
        q = self.q_proj(x)  # [B, T, C]
        k = self.k_proj(x)
        v = self.v_proj(x)

        # 2. 拆成多头
        q = rearrange(q, 'b t (h d) -> b h t d', h=self.num_heads)
        k = rearrange(k, 'b t (h d) -> b h t d', h=self.num_heads)
        v = rearrange(v, 'b t (h d) -> b h t d', h=self.num_heads)

        # 3. 计算注意力分数
        scores = torch.einsum('bhtd,bhsd->bhts', q, k) * self.scale
        # scores: [B, H, T, T]

        # 4. 因果掩码
        mask = torch.triu(torch.ones(T, T, dtype=torch.bool), diagonal=1)
        scores = scores.masked_fill(mask, float('-inf'))

        # 5. Softmax + 加权
        attn = F.softmax(scores, dim=-1)
        o = torch.einsum('bhts,bhsd->bhtd', attn, v)

        # 6. 合并头 + 输出投影
        o = rearrange(o, 'b h t d -> b t (h d)')
        return self.o_proj(o)
对比 GDN：GDN 的层结构几乎完全一样——也有 q_proj/k_proj/v_proj/o_proj 和多头拆分。 唯一不同的是第3-5步（注意力计算）被替换为 chunk_gated_delta_rule(q, k, v, ...)。 理解了这个结构，GDN 的层代码你就能读懂 80%。

测验

问题 1：注意力公式中除以 √d 的目的是什么？
  • 减少参数量
  • 防止点积过大导致 softmax 饱和
  • 加速矩阵乘法
  • 实现因果掩码
问题 2：因果掩码的作用是什么？
  • 减少计算量
  • 防止过拟合
  • 让每个 token 只能看到它之前的 token
  • 对注意力分数做归一化
问题 3：KV Cache 的主要问题是什么？
  • 计算太慢
  • 显存随序列长度线性增长 O(N×d)
  • 精度不够
  • 无法并行化
