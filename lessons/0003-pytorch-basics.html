PyTorch 核心

GDN 的参考实现用 PyTorch 编写。你需要理解 Tensor、autograd、nn.Module 和 einsum—— 这些是读懂 GDN 代码、编写验证脚本的必备工具。

Tensor：多维数组

Tensor 是 PyTorch 的基本数据结构，本质是带设备和梯度追踪的多维数组。

import torch

# 创建 tensor
x = torch.randn(32, 128, 512)  # [batch, seq_len, hidden_size]

# 关键属性
x.shape    # torch.Size([32, 128, 512]) — 形状
x.dtype    # torch.float32 — 数据类型
x.device   # device(type='cpu') — 在哪个设备上

# 常用操作
y = x @ x.transpose(-1, -2)  # 矩阵乘法
z = x.reshape(32, 128, 16, 32)  # 改形状
w = x.transpose(1, 2)  # 交换维度
GDN 中的 tensor 形状约定：
  • q, k: [B, T, H, K] — batch, seq_len, num_heads, head_dim
  • v: [B, T, H, V] — V 可以不等于 K
  • beta, g: [B, T, H] — 每个头一个标量
  • state S: [B, H, K, V] — 每个头一个矩阵
记住这个形状约定，后面看代码会轻松很多。

Autograd：自动求导

PyTorch 的 autograd 自动追踪 tensor 上的操作，构建计算图，并在调用 backward() 时自动做反向传播。

import torch

# 需要 requires_grad=True 才会追踪梯度
W = torch.randn(4, 3, requires_grad=True)
x = torch.randn(2, 4)
y = torch.randn(2, 3)

# 前向传播（autograd 自动记录操作）
z = x @ W           # 矩阵乘法
loss = ((z - y) ** 2).mean()

# 反向传播（自动！）
loss.backward()

# 梯度在 W.grad 中
print(W.grad)       # shape [4, 3]，与 W 同形状
# tensor([[ 0.12, -0.05,  0.33],
#         [-0.08,  0.21, -0.14],
#         [ 0.45,  0.02, -0.27],
#         [-0.11,  0.09,  0.18]])
autograd 的局限：它只能对 PyTorch 内置操作自动求导。 GDN 的核心 kernel chunk_gated_delta_rule 是一个自定义操作， PyTorch 不知道内部做了什么。所以 GDN 的 kernel 必须手动实现前向和反向， 通过 torch.autograd.Function 注册。

nn.Module：层和模型的基类

所有神经网络层都继承 nn.Module。你需要知道三件事：

import torch.nn as nn

class MyLayer(nn.Module):
    def __init__(self, hidden_size):
        super().__init__()
        # 1. 在 __init__ 中定义参数和子模块
        self.proj = nn.Linear(hidden_size, hidden_size, bias=False)
        self.norm = nn.RMSNorm(hidden_size)

    def forward(self, x):
        # 2. 在 forward 中定义前向计算
        return self.norm(self.proj(x))

# 3. 参数自动注册，优化器可以找到它们
layer = MyLayer(512)
optimizer = torch.optim.Adam(layer.parameters(), lr=1e-4)

GDN 的层定义正是这个结构，看真实代码片段：

class GatedDeltaNet(nn.Module):
    def __init__(self, hidden_size=2048, ...):
        super().__init__()
        self.q_proj = nn.Linear(hidden_size, key_dim, bias=False)
        self.k_proj = nn.Linear(hidden_size, key_dim, bias=False)
        self.v_proj = nn.Linear(hidden_size, value_dim, bias=False)
        # ... 更多参数

    def forward(self, hidden_states):
        q = self.q_proj(hidden_states)  # 投影
        k = self.k_proj(hidden_states)
        v = self.v_proj(hidden_states)

einsum：用公式做矩阵运算

torch.einsum 用下标公式表示矩阵运算，比 @ 和 bmm 更直观。 GDN 代码中大量使用，必须能读懂。

基本语法

# 矩阵乘法: C = A @ B
C = torch.einsum('ij,jk->ik', A, B)

# 转置: B = A^T
B = torch.einsum('ij->ji', A)

# 内积: c = a · b
c = torch.einsum('i,i->', a, b)

# 外积: M = a ⊗ b (产生矩阵!)
M = torch.einsum('i,j->ij', a, b)
外积的 einsum 表示：GDN 的状态更新是 S += k ⊗ v， 在 einsum 中就是 'k,v->kv'。这就是你需要在昇腾上实现的核心操作之一。

带批量和多头的 einsum（GDN 实际使用）

# GDN naive 实现中的核心计算:
# q: [B, H, K], S: [B, H, K, V] → o: [B, H, V]
o = torch.einsum('bhd,bhdm->bhm', b_q, h)
#  ^^^         ^^^^^^^^^^   ^^^^
#  q的维度      S的维度       输出维度
# b=batch, h=head, d=key_dim, m=value_dim

# 状态更新: h += k ⊗ v
# k: [B, H, K], v: [B, H, V] → h: [B, H, K, V]
h = h + torch.einsum('bhd,bhm->bhdm', b_k, b_v)

einsum 速查表

操作 einsum 等价代码
矩阵乘法 'ij,jk->ik' A @ B
外积 'i,j->ij' a.unsqueeze(1) * b.unsqueeze(0)
批量矩阵乘 'bhd,bhdm->bhm' (A @ B).diagonal(...)
转置 'ij->ji' A.T
逐元素乘 'ij,ij->ij' A * B

rearrange：einops 的维度变换

GDN 代码大量使用 einops.rearrange，它是 reshape + transpose 的人类可读版本：

from einops import rearrange

# 把最后一维拆成 (heads, head_dim)
# [B, T, hidden] → [B, T, H, D]
q = rearrange(q, '... (h d) -> ... h d', h=num_heads)
#                        ^^^^^^    ^^^^
#                        合并的维  拆成两个

# 反过来合并
# [B, T, H, D] → [B, T, hidden]
o = rearrange(o, 'b t h d -> b t (h d)')
einops 规则：左边描述输入形状，右边描述输出形状。 ... 匹配任意不关心的维度。(h d) 表示两个维度被合并，拆分时指定其中一个的大小。

综合：读懂 GDN naive 代码

现在你已经有了所有工具。试着读这段 GDN 的参考实现（来自 fla/ops/gated_delta_rule/naive.py）：

def naive_recurrent_gated_delta_rule(q, k, v, beta, g, scale=None, initial_state=None):
    # q: [B, T, H, K],  k: [B, T, H, K],  v: [B, T, H, V]
    # beta: [B, T, H],  g: [B, T, H]
    q, k, v, beta, g = map(
        lambda x: x.transpose(1, 2).contiguous().to(torch.float32),
        [q, k, v, beta, g]
    )  # → [B, H, T, K] 格式，方便循环

    B, H, T, K = k.shape
    V = v.shape[-1]
    h = torch.zeros(B, H, K, V)  # 状态矩阵 S

    for i in range(T):
        b_q = q[:, :, i]     # [B, H, K]
        b_k = k[:, :, i]     # [B, H, K]
        b_v = v[:, :, i]     # [B, H, V]

        # 1. 门控：旧状态乘以衰减因子
        h = h * g[:, :, i].exp()[..., None, None]

        # 2. Delta 规则：计算修正值
        b_v = b_v - (h * b_k[..., None]).sum(-2)

        # 3. 用 beta 缩放修正值
        b_v = b_v * beta[:, :, i][..., None]

        # 4. 写入状态
        h = h + b_k.unsqueeze(-1) * b_v.unsqueeze(-2)  # 外积!

        # 5. 读取输出
        o[:, :, i] = torch.einsum('bhd,bhdm->bhm', b_q, h)

    return o, h
你能读懂多少？ 如果大部分能看懂，说明你准备好了。 重点关注：
  • h * g[..., None, None] — 门控（第3课会详细讲）
  • (h * b_k[..., None]).sum(-2) — 用 k 查询当前状态
  • b_k.unsqueeze(-1) * b_v.unsqueeze(-2) — 外积 k⊗v
  • einsum('bhd,bhdm->bhm', b_q, h) — 用 q 读取状态
这些操作的具体含义在后续 GDN 课程中会逐一展开。

测验

问题 1：einsum 'i,j->ij' 表示什么操作？
  • 内积
  • 外积
  • 矩阵乘法
  • 逐元素乘
问题 2：rearrange 'b t (h d) -> b t h d' 做了什么？
  • 把 h 和 d 合并成一维
  • 把最后一维拆分成 h 和 d 两个维度
  • 转置 b 和 t 维度
  • 删除 h 维度
问题 3：为什么 GDN kernel 不能用 autograd 自动求导？
  • 因为 GDN 使用 float32
  • 因为 einsum 不支持自动求导
  • 因为 kernel 是自定义操作，PyTorch 不知道内部计算步骤
  • 因为 GDN 的梯度不存在
